# Flujo de trabajo del equipo

## Flujo propuesto (versión inicial)

1. Clonar el repositorio.
2. Revisar estado con `git status`.
3. Crear rama por tarea o misión.
4. Editar archivos asignados.
5. Revisar cambios con `git diff`.
6. Agregar con `git add`.
7. Confirmar con `git commit`.
8. Integrar con `git merge`.
HOLA HOLA HOLA

## Acuerdos de edición

- Mantener mensajes de commit breves y descriptivos.
- No editar todo el documento si la tarea pide una sección.
- Resolver conflictos leyendo ambas versiones antes de confirmar.
